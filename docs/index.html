Singular queueing to ease network usage.

Installation

Simply put the eph-queue folder to your resources directory and append it to your server configuration file.

Resource Structure

  1. Below is the file structure in case you are interested:
    • eph-queue - Root folder for the resource
      • fxmanifest.lua - Script's manifest file.
      • server.js - Server side code to handle queueing and refreshing.
      • client.lua - An event adapter to tell the server when the player fully loaded the game.

Code Structure & Developer Usage

eph-queue targets to be integrateable to all other scripts. You can use the exports below to control the services eph-queue provides to you.

local queueSize = exports["eph-queue"]:queueSize() --will return how many players are in the queue
print(queueSize) --18
              
exports["eph-queue"]:clearQueue() --will default return `0` (new size of the queue after its cleared)
exports["eph-queue"]:toggleQueue() --will return the new state of the queue (true / false) 
exports["eph-queue"]:queueState() -- queue is enabled : returns true
                                   --queue is disabled : returns false 
exports["eph-queue"]:isInQueue(playerName) --true / false 
local queueplayers = exports["eph-queue"]:getQueuedPlayers() --gets player USER NAMES in queue
print(queueplayers) --"{'user1', 'user2', 'user3', 'user4', 'user5'}"

Commands

 togglequeue | turns queue on or off 
 queuesize | prints the queue length
 queuestate | prints whether the queue is enabled or disabled
 clearqueue | clears the queue
